BUG 2: Provider dispatcher runs ALL carriers for one order
═══════════════════════════════════════════════════════════════════════
Logs show DIGYLOG-FINAL, AMEEX-PRE-SHIP, AND the express_coursier_cities error ALL for the same order. This means the bulk-ship dispatcher is NOT properly branching on the selected provider — it's executing all carrier code paths.

Open server/routes.ts → /api/orders/bulk-ship handler and server/services/carrier-service.ts → shipOrderToCarrier.

The dispatcher MUST be a strict if/else (or switch) that runs ONLY the selected carrier:

  async function shipOrderToCarrier(order, creds, options = {}) {
    const provider = (options.provider || creds.carrierName || "").toLowerCase().trim();

    switch (provider) {
      case "digylog":
        return await shipOrderToDigylog(order, creds, options);
      case "ameex":
        return await shipOrderToAmeex(order, creds, options);
      case "expresscoursier":
        return await shipOrderToExpressCoursier(order, creds, options);
      default:
        throw new Error(`Provider "${provider}" non supporté.`);
    }
  }

NO fallthrough. NO running multiple branches. Each provider path must be isolated.

If the existing code has Digylog/Ameex logic OUTSIDE the switch (running unconditionally as "pre-ship" steps), MOVE that logic INSIDE its own case. The [AMEEX-PRE-SHIP] log line running for an Express Coursier order is a smoking gun — Ameex prep code is running unconditionally.

Search for [DIGYLOG-FINAL] and [AMEEX-PRE-SHIP] log strings and trace what wraps them. Wrap each carrier's prep + ship logic inside an `if (provider === "X")` block.
