## STEP 1 — Correct column detection (never grab a metric column)
Spend column must resolve to Facebook's `Amount spent (USD)`, never `Cost per results`/CPM/CPC.
```ts
function findSpendColumn(headers: string[]): string | null {
  const n = (h: string) => h.toLowerCase().trim();
  let h = headers.find(x => n(x).includes("amount spent")); if (h) return h;
  h = headers.find(x => n(x).includes("montant dépensé") || n(x).includes("montant depense")); if (h) return h;
  h = headers.find(x => /total\s*(cost|spent|spend)/.test(n(x))); if (h) return h;
  h = headers.find(x => ["spend","dépenses","depenses"].includes(n(x))); if (h) return h;
  h = headers.find(x => { const s = n(x); return s.includes("cost") && !s.includes("per") && !s.includes("cpm") && !s.includes("cpc"); });
  return h ?? null;
}
```
Always show the detected Campagne / Montant / Date columns as dropdowns the user can override; recompute amounts on change. Apply the same `per`/`cpm`/`cpc` exclusion so campaign/date detection never grabs a metric column.

## STEP 5 — Aggregate spend PER PRODUCT (sum campaigns → same product)
The ad-spend table has no campaign column, so consolidate. On import:
1. Keep only rows with a product selected (skip "Aucun produit").
2. Group by `productId` (within same `source` + `date`), **sum** the DH amounts.
3. Insert ONE ad-spend entry per product: `{ source, date, productId, magasinId, amount=summedDh }` storing `Math.round(summedDh*100)` centimes.
Example: `mini 9tara`→قطرات العسل والزيت (2000) + `mini 9tara news`→قطرات العسل والزيت (1000) ⇒ one entry of **3000 DH** for قطرات العسل والزيت.
Also upsert each confirmed `(storeId, normalize(campaignName), productId)` into `ad_campaign_product_map` (skip duplicates) so future imports auto-link.
