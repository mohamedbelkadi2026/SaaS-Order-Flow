Two-part task on this repo. SOURCE = "digylog_webhook" orders show "-" in the PRODUIT 
column; SOURCE = "shopify" orders show the product correctly. I want the product name 
to keep showing going forward, AND to recover it for existing broken orders where 
possible.

ROOT CAUSE (already diagnosed — do NOT re-investigate):
In server/routes.ts, processCarrierWebhook() has an auto-create fallback: when a 
Digylog status webhook arrives with a tracking number that matches NO existing order 
(by tracking_number, then order_number, then cross-store tracking), it calls 
storage.createOrderFromCarrier(), creating a new order with source "<carrier>_webhook". 
Digylog's fetchOrderDetails() (server/services/carrier-service.ts) returns 
name/phone/address/city/price but NO product name, so these orders have empty 
rawProductName and no order_items → "-". The product name can ONLY be preserved by 
routing the Digylog status onto the existing (Shopify) order that already has it — 
never by reading it from Digylog. The frontend product column resolves as:
order.rawProductName || items[0]?.rawProductName || items[0]?.product?.name || '-'.

=========================================================
1) BEFORE the existing auto-create block runs, add a phone-number match fallback:
   - Get the customer phone from the webhook payload; if missing, use the phone from

     non-digits) using the SAME normalization helper the app already uses elsewhere

3) Do NOT change Shopify creation, and do NOT attempt to read a product name from
