FEATURE — Add full SEO meta tags + favicon set so the platform has proper branding when shared / searched
My problem
When someone shares tajergrow.com on WhatsApp / Facebook / Google search, they see:

No site title ("tajergrow.com" raw)
No description
No icon / logo
No preview image

I want the platform to show up properly with logo, name, description, and preview image whenever someone:

Bookmarks the site
Shares it on social media
Searches it on Google
Pins it to mobile home screen


Step 1 — Generate the favicon set
The user has these 3 options. Pick the one that matches what was selected:
Option A — User has a logo file (PNG/SVG)
Place the logo at client/public/logo-master.png (1024×1024 PNG, transparent background). Then generate every needed size from it.
Install once:
bashnpm install -D sharp
Create a script at scripts/generate-favicons.mjs:
jsimport sharp from 'sharp';
import { mkdirSync } from 'fs';
import { join } from 'path';

const SRC  = 'client/public/logo-master.png';
const DEST = 'client/public';
mkdirSync(DEST, { recursive: true });

const sizes = [
  { name: 'favicon-16x16.png',      size: 16  },
  { name: 'favicon-32x32.png',      size: 32  },
  { name: 'favicon-48x48.png',      size: 48  },
  { name: 'favicon.png',            size: 192 },        // legacy, overwrites existing
  { name: 'apple-touch-icon.png',   size: 180 },        // iOS home screen
  { name: 'android-chrome-192.png', size: 192 },
  { name: 'android-chrome-512.png', size: 512 },
  { name: 'og-image.png',           size: 1200, height: 630 },  // Facebook / WhatsApp share preview
];

for (const { name, size, height } of sizes) {
  await sharp(SRC)
    .resize(size, height || size, { fit: 'contain', background: { r: 30, g: 27, b: 75, alpha: 1 } })  // navy bg matches your brand
    .png()
    .toFile(join(DEST, name));
  console.log(`✓ ${name}`);
}
Run once:
bashnode scripts/generate-favicons.mjs
This generates 8 image files in client/public/. Commit them.

Option C — User wants simple text "T" icon
Skip image generation. Use a pure SVG favicon with the letter "T" on the brand background:
Create client/public/favicon.svg:
svg<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 64 64">
  <rect width="64" height="64" rx="14" fill="#1E1B4B"/>
  <text x="32" y="44" font-family="Inter, sans-serif" font-size="40" font-weight="800" fill="#C5A059" text-anchor="middle">T</text>
</svg>
For Apple/Android, create client/public/apple-touch-icon.png from the same SVG using any online converter (or skip — SVG works for most modern browsers).

Step 4 — robots.txt and sitemap.xml for Google indexing
Create client/public/robots.txt:
User-agent: *
Allow: /
Disallow: /api/
Disallow: /admin
Disallow: /super-admin
Disallow: /checkout

Sitemap: https://www.tajergrow.com/sitemap.xml
Create client/public/sitemap.xml:
xml<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://www.tajergrow.com/</loc>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://www.tajergrow.com/billing</loc>
    <changefreq>monthly</changefreq>
    <priority>0.7</priority>
  </url>
  <url>
    <loc>https://www.tajergrow.com/terms</loc>
    <changefreq>yearly</changefreq>
    <priority>0.3</priority>
  </url>
</urlset>

Step 5 — Submit to Google Search Console
After deploy:

Go to https://search.google.com/search-console
