FEATURE — Product inventory shows full historical performance + restock-aware Reçu calculation + per-product stats modal
My problem
When I add a product to inventory and then do a restock 2 days/weeks later, I want the inventory page to show me everything I need to decide whether to keep buying this product:

Total received (cumulative across all restocks)
Total delivered
Total confirmed
Confirmation rate (lifetime)
Delivery rate (lifetime)
Currently in transit
Currently available

Today the page shows lifetime stats correctly for products with continuous orders, but two things are missing:

Restock history tracking — I can't see when I last added stock and how much I added each time
A detailed per-product modal — clicking a product should show me its full performance: orders timeline, top cities, refusal reasons, current open orders, etc. So I know whether to reorder or kill the product.

What to build
Part 1 — Stock movement tracking (so "Reçu" reflects actual restocks, not just current_stock + delivered)
Today the Reçu number is computed as available + sortie. That works if delivered orders never get returned and stock never gets manually adjusted. The moment you do a manual restock, the formula becomes wrong (e.g. you added 50 units but Reçu jumped by 50 too, even though you only "received" 50 once).
I want a real ledger.
Part 2 — Per-product details modal
Clicking a product row opens a side panel/modal with:

Lifetime totals (already computed, just shown bigger)
Orders by status (pie chart or simple bars)
Last 30 orders for this product (table)
Stock movement history (every restock + every delivery)
Top 5 cities for this product
Top 5 refusal reasons for this product (from comments / status notes)


Step 1 — Migration migrations/0004_stock_movements.sql
sql-- ============================================================================
-- 0004_stock_movements
-- Ledger of every stock change. The "Reçu" column on the inventory page is
-- computed from this ledger (sum of all 'restock' rows) instead of being
-- inferred from current_stock + delivered_count.
-- ============================================================================

CREATE TABLE IF NOT EXISTS public.stock_movements (
  id           serial PRIMARY KEY,
  store_id     integer NOT NULL REFERENCES public.stores(id) ON DELETE CASCADE,
  product_id   integer NOT NULL REFERENCES public.products(id) ON DELETE CASCADE,
  variant_id   integer,
  type         text    NOT NULL CHECK (type IN (
                  'restock',         -- manual + initial stock add
                  'delivered',       -- order status → delivered (auto)
                  'returned',        -- carrier returned (auto)
                  'adjustment',      -- manual correction (positive or negative)
                  'reservation',     -- order confirmed, stock allocated
                  'release'          -- order cancelled, stock freed
                )),
  quantity     integer NOT NULL,    -- positive = stock in, negative = stock out
  reason       text,                -- free-text note ("Inventory recount", "Damaged", etc.)
  order_id     integer REFERENCES public.orders(id) ON DELETE SET NULL,
  user_id      integer REFERENCES public.users(id),
  created_at   timestamp NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_stock_movements_product
  ON public.stock_movements (product_id, created_at DESC);

CREATE INDEX IF NOT EXISTS idx_stock_movements_store
  ON public.stock_movements (store_id, created_at DESC);

-- ── Backfill the ledger from existing data ────────────────────────────────
-- One 'restock' row per product representing the initial stock that was
-- entered before this ledger existed. Quantity = current_stock + delivered_count.
-- This makes the new Reçu calculation match the old behavior on day 1.
INSERT INTO public.stock_movements (store_id, product_id, type, quantity, reason, created_at)
SELECT
  p.store_id,
  p.id,
  'restock',
  GREATEST(
    p.stock + COALESCE((
      SELECT SUM(oi.quantity)
      FROM public.order_items oi
      INNER JOIN public.orders o ON o.id = oi.order_id
      WHERE oi.product_id = p.id AND o.status = 'delivered'
    ), 0),
    p.stock
  ),
  'Initial backfill — pre-ledger inventory',
  COALESCE(p.created_at, now())
FROM public.products p
WHERE NOT EXISTS (
  SELECT 1 FROM public.stock_movements sm WHERE sm.product_id = p.id AND sm.type = 'restock'
);
This migration creates the ledger and backfills one row per existing product so the inventory page numbers stay identical the moment you deploy. Going forward, every restock and every delivery writes a new row.

Step 4 — server/storage.ts: rewrite getInventoryStats to use the ledger
Replace the old Reçu = available + sortie formula with a real query against the ledger.
tsasync getInventoryStats(storeId: number, options?: { dateFrom?: string; dateTo?: string }): Promise<any> {
  const allProducts = await db.select().from(products).where(eq(products.storeId, storeId));
  const allVariants = await db.select().from(productVariants).where(eq(productVariants.storeId, storeId));

  // (existing aggregates: totalProducts, totalQuantity, lowStock, outOfStock, newProducts)
  // ... same as before ...

  // Pull all movements for these products in one query
  const productIds = allProducts.map(p => p.id);
  const movements = productIds.length === 0
    ? []
    : await db.select().from(stockMovements)
        .where(and(
          eq(stockMovements.storeId, storeId),
          inArray(stockMovements.productId, productIds),
        ));

  // Group movements by product
  const movementsByProduct = new Map<number, typeof movements>();
  for (const m of movements) {
    if (!movementsByProduct.has(m.productId)) movementsByProduct.set(m.productId, []);
    movementsByProduct.get(m.productId)!.push(m);
  }

  const productStats = [];
  for (const p of allProducts) {
    const variants = allVariants.filter(v => v.productId === p.id);
    const totalStock = p.stock + variants.reduce((s, v) => s + v.stock, 0);
    const productMovements = movementsByProduct.get(p.id) || [];

    // Reçu = sum of every 'restock' (lifetime, never decreases)
    const recu = productMovements
      .filter(m => m.type === 'restock')
      .reduce((s, m) => s + m.quantity, 0);

    // Sortie = sum of |delivered|
    const sortie = Math.abs(productMovements
      .filter(m => m.type === 'delivered')
      .reduce((s, m) => s + m.quantity, 0));

    // Returns = sum of |returned|
    const returned = productMovements
      .filter(m => m.type === 'returned')
      .reduce((s, m) => s + m.quantity, 0);

    // Last restock info (for tooltip)
    const lastRestock = productMovements
      .filter(m => m.type === 'restock')
      .sort((a, b) => new Date(b.createdAt).getTime() - new Date(a.createdAt).getTime())[0];

    // Existing in-transit / confirmation calculations stay (use orders directly)
    // ... existing INVENTORY_CONFIRMED / inTransitItems / totalOrderItems queries ...

    const confirmRate = totalOrdered > 0 ? Math.round(totalConfirmedQty / totalOrdered * 100) : 0;
    const deliverRate = totalConfirmedQty > 0 ? Math.round(sortie / totalConfirmedQty * 100) : 0;

    productStats.push({
      ...p,
      totalStock,
      recu,                          // ← from ledger (true cumulative)
      sortie,
      returned,
      inTransit,
      totalOrdered,
      confirmRate,
      deliverRate,
      lastRestockAt:  lastRestock?.createdAt ?? null,
      lastRestockQty: lastRestock?.quantity ?? null,
    });
  }

  return { totalProducts, totalQuantity, lowStock, outOfStock, newProducts, productStats };
}
Step 5 — New endpoint for per-product detailed stats
server/routes.ts:
tsapp.get("/api/products/:id/insights", requireAuth, async (req, res) => {
  const productId = Number(req.params.id);
  const storeId = req.user!.storeId!;

  // Verify ownership
  const [product] = await db.select().from(products)
    .where(and(eq(products.id, productId), eq(products.storeId, storeId)));
  if (!product) return res.status(404).json({ message: "Produit introuvable" });

  // 1. Stock movements (ledger)
  const movements = await db.select().from(stockMovements)
    .where(and(eq(stockMovements.productId, productId), eq(stockMovements.storeId, storeId)))
    .orderBy(desc(stockMovements.createdAt))
    .limit(100);

  // 2. Last 30 orders for this product
  const recentOrderItems = await db.select({
    orderId:        orderItems.orderId,
    quantity:       orderItems.quantity,
    price:          orderItems.price,
    orderNumber:    orders.orderNumber,
    customerName:   orders.customerName,
    customerCity:   orders.customerCity,
    status:         orders.status,
    totalPrice:     orders.totalPrice,
    createdAt:      orders.createdAt,
    deliveredAt:    orders.deliveredAt,
  })
    .from(orderItems)
    .innerJoin(orders, eq(orderItems.orderId, orders.id))
    .where(and(
      eq(orderItems.productId, productId),
      eq(orders.storeId, storeId),
    ))
    .orderBy(desc(orders.createdAt))
    .limit(30);

  // 3. Aggregates: top cities (delivered), refusal reasons
  const allOrdersForProduct = await db.select({
    city:    orders.customerCity,
    status:  orders.status,
    comment: orders.comment,
    qty:     orderItems.quantity,
  })
    .from(orderItems)
    .innerJoin(orders, eq(orderItems.orderId, orders.id))
    .where(and(
      eq(orderItems.productId, productId),
      eq(orders.storeId, storeId),
    ));

  // Top cities (by delivered count)
  const cityMap: Record<string, number> = {};
  allOrdersForProduct.filter(o => o.status === 'delivered').forEach(o => {
    const c = (o.city || 'Inconnu').trim();
    cityMap[c] = (cityMap[c] || 0) + (o.qty || 1);
  });
  const topCities = Object.entries(cityMap)
    .map(([city, count]) => ({ city, count }))
    .sort((a, b) => b.count - a.count)
    .slice(0, 5);

  // Status breakdown (for pie chart)
  const statusMap: Record<string, number> = {};
  allOrdersForProduct.forEach(o => {
    statusMap[o.status] = (statusMap[o.status] || 0) + 1;
  });

  // Refusal patterns (rough — based on order comment substrings)
  const refusalReasons: Record<string, number> = {};
  allOrdersForProduct
    .filter(o => o.status === 'refused' || o.status?.startsWith('Annulé'))
    .forEach(o => {
      const reason = (o.comment || 'Sans raison').slice(0, 60);
      refusalReasons[reason] = (refusalReasons[reason] || 0) + 1;
    });
  const topRefusalReasons = Object.entries(refusalReasons)
    .map(([reason, count]) => ({ reason, count }))
    .sort((a, b) => b.count - a.count)
    .slice(0, 5);

  res.json({
    product,
    movements,
    recentOrders: recentOrderItems,
    topCities,
    statusBreakdown: statusMap,
    topRefusalReasons,
  });
});

tsapp.post("/api/products/:id/restock", requireAdmin, async (req, res) => {
  const productId = Number(req.params.id);
  const storeId = req.user!.storeId!;
  const { quantity, reason } = req.body;

  if (!Number.isInteger(quantity) || quantity <= 0) {
    return res.status(400).json({ message: "quantity doit être un entier positif" });
  }

  // Verify product belongs to store
  const [product] = await db.select().from(products)
    .where(and(eq(products.id, productId), eq(products.storeId, storeId)));
  if (!product) return res.status(404).json({ message: "Produit introuvable" });

  await db.transaction(async (tx) => {
    await tx.update(products)
      .set({ stock: sql`${products.stock} + ${quantity}` })
      .where(eq(products.id, productId));

      storeId,
      productId,
      type: 'restock',
      quantity,
      reason: reason || 'Manual restock',
