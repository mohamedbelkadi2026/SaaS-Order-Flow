🐛 FIX — Quantity field forces user to use arrows; can't type freelyMy problemIn the order edit modal, the quantity field is <Input type="number" min={1} value={qty}>. Two issues:
Spinner arrows are intrusive — user must click ↑/↓ to change quantity 1 → 2 → 3, which is painful for big orders (e.g. quantity 50).
onChange does parseInt(...) || 1 — when the user clears the field to retype (e.g. to enter "12" they need to first clear "1"), the value snaps back to 1 the moment the field is empty. They can't even start typing the new number.
Same issue exists on the price field (line 142): Math.round(parseFloat(e.target.value) * 100) chokes on a temporarily empty input.FixIn client/src/components/order-details-modal.tsx, replace the quantity input (lines 149-156) with a typeable input that holds local string state during editing:tsx{/* Quantity input — typeable, no spinner arrows, accepts free editing */}
<Input
  type="text"
  inputMode="numeric"
  pattern="[0-9]*"
  value={qty === 0 ? '' : String(qty)}
  onFocus={(e) => e.target.select()}
  onChange={(e) => {
    const raw = e.target.value.replace(/[^\d]/g, '');
    // Allow empty during typing; commit 0 to local state
    // and let the parent decide what to do on blur if needed.
    if (raw === '') {
      onChange(item.id, "quantity", 0);
      return;
    }
    const n = parseInt(raw, 10);
    if (Number.isFinite(n) && n >= 0) {
      onChange(item.id, "quantity", n);
    }
  }}
  onBlur={(e) => {
    // On blur, if empty or 0, snap to 1 so we never persist a 0-qty item
    if (qty === 0 || qty < 1) {
      onChange(item.id, "quantity", 1);
    }
  }}
  className="w-16 text-sm text-center font-semibold h-8 rounded-lg
             [appearance:textfield]
             [&::-webkit-outer-spin-button]:appearance-none
             [&::-webkit-inner-spin-button]:appearance-none"
  style={{ color: NAVY }}
  data-testid="input-item-quantity"
/>Key differences:

type="text" + inputMode="numeric" + pattern="[0-9]*" → no native spinner, but mobile keyboards still show numeric pad
value={qty === 0 ? '' : String(qty)} → shows empty during editing, not "0"
onChange strips non-digits and accepts 0 temporarily so user can clear & retype
onBlur enforces minimum 1 (parents never see qty=0 persisted)
The Tailwind classes hide spinner buttons even if a browser ignores type="text"
onFocus={e.target.select()} selects all on focus → user can immediately start typing the new number without clearing first
Width w-16 (was w-14) gives room for 2-3 digit numbers
Step 2 — Same fix for the price field (line 139-147)Apply the same pattern:tsx<Input
  type="text"
  inputMode="decimal"
  value={priceVal === 0 ? '' : priceVal.toFixed(2)}
  onFocus={(e) => e.target.select()}
  onChange={(e) => {
    const raw = e.target.value.replace(/[^\d.,]/g, '').replace(',', '.');
    if (raw === '') {
      onChange(item.id, "price", 0);
      return;
    }
    const f = parseFloat(raw);
    if (Number.isFinite(f) && f >= 0) {
      onChange(item.id, "price", Math.round(f * 100));  // store cents
    }
  }}
  onBlur={() => {
    // No snap-to-min on price — user might legitimately want 0 for free items / promos
  }}
  className="w-20 text-sm text-right font-semibold h-8 rounded-lg
             [appearance:textfield]
             [&::-webkit-outer-spin-button]:appearance-none
             [&::-webkit-inner-spin-button]:appearance-none"
  style={{ color: NAVY }}
  data-testid="input-item-price"
