ne correspondent pas au Dashboard (289 commandes / 174 livrées) ni à
l'onglet "Par Plateforme" du même Profit Analyzer (qui donne bien 174 en
sommant les livrées par plateforme).

CAUSE RACINE (confirmée dans le code)
- server/services/profit.ts, fonction computeProfitability() : le total
  correct existe DÉJÀ, calculé à partir des commandes elles-mêmes
  (dédupliqué, une commande = une unité) :
    totals.totalOrders     = storeOrders.length;
    totals.deliveredOrders = storeOrders.filter(o => isDeliveredStatus(o.status)).length;
  → C'est CE nombre qui doit s'afficher (il correspond exactement au Dashboard).

- server/routes.ts, route GET /api/products/profitability (~ligne 7339-7346) :
  la réponse JSON ne renvoie PAS ce champ `totals` :
    res.json({ products: result.products, platforms: result.platforms, globalAdSpend: result.globalAdSpend });
  → `totals` est calculé côté serveur mais jamais envoyé au frontend.

- client/src/pages/profit-analyzer.tsx (~ligne 356-360) : en l'absence de
  `totals`, le frontend recalcule lui-même en sommant PAR PRODUIT :
    const liveTotalOrders    = liveProducts.reduce((s, p) => s + p.totalOrders, 0);
    const liveTotalDelivered = liveProducts.reduce((s, p) => s + p.deliveredOrders, 0);
  → Une commande contenant 2 produits différents (ex: sandale + ceinture)
    est comptée une fois dans les stats du produit A ET une fois dans les
    stats du produit B → total gonflé (305 au lieu de 289, 188 au lieu de 174).
    L'onglet "Par Plateforme" n'a pas ce problème car il regroupe par
    commande (une commande = une plateforme), pas par produit.

FIX A — server/routes.ts, GET /api/products/profitability

Renvoyer aussi `totals` :

```ts
app.get("/api/products/profitability", requireAuth, async (req, res) => {
  try {
    const storeId = req.user!.storeId!;
    const { dateFrom, dateTo, dateRange } = req.query as Record<string, string>;
    const result = await computeProfitability(storeId, { dateFrom, dateTo, dateRange });
    res.json({ products: result.products, platforms: result.platforms, totals: result.totals, globalAdSpend: result.globalAdSpend });
  } catch (err) {
    throw err;
  }
});
```
