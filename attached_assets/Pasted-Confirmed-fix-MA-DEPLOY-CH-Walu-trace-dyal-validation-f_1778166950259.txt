URGENT BUG FIX — Ameex rejecting orders for missing customerName
Symptom
Server logs show repeatedly:
[CARRIER→AMEEX][#5289] ❌ Ameex n'a pas retourné de numéro de suivi.
Raw: {"login":"success","api":{"type":"error","msg":"Destinataire: est obligatoire"}}
Ameex is correctly rejecting orders because the destinataire (customer name) field is empty or contains only invisible characters (RTL marks, zero-width spaces from Shopify imports). The platform sends the request anyway and surfaces a confusing error.
This affects orders imported from Shopify/WooCommerce where customer names sometimes contain \u200E (LRM), \u200F (RLM), \u200B (ZWSP), \uFEFF (BOM), etc. The string LOOKS non-empty in the database but cleans to empty.

Change A — Add helper function above buildAmeexPayload (which is around line 365):
ts/**
 * Strip invisible Unicode characters that often hide in customer-facing fields
 * imported from Shopify/WooCommerce stores. These characters render as nothing
 * in the UI but cause server-side validation failures on third-party APIs.
 */
function cleanText(s: any): string {
  return String(s ?? '')
    .replace(/[\u200B-\u200F\u202A-\u202E\u2066-\u2069\uFEFF\u00A0]/g, '')
    .replace(/\s+/g, ' ')
    .trim();
}

/**
 * Validate Ameex-mandatory fields BEFORE making the API call.
 * Returns a list of missing/invalid fields, or empty array if OK.
 */
function validateAmeexInput(input: CarrierShipInput): string[] {
  const issues: string[] = [];
  const name    = cleanText(input.customerName);
  const phone   = cleanText(input.phone).replace(/\D/g, '');
  const city    = cleanText(input.city);
  const address = cleanText(input.address);
  const product = cleanText(input.productName);

  if (!name || name.length < 2)       issues.push('Nom du destinataire manquant ou trop court');
  if (!phone || phone.length < 8)     issues.push('Téléphone manquant ou invalide');
  if (!city)                          issues.push('Ville manquante');
  if (!address && !city)              issues.push('Adresse manquante');
  if (!product)                       issues.push('Nom du produit manquant');
  if (!input.totalPrice || input.totalPrice <= 0) issues.push('Prix total manquant ou nul');

  return issues;
}
