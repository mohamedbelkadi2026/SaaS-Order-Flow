ÉTAPE 1 — Renforcer normStr() : le nom catalogue contient un ESPACE DOUBLE

Dans "Modifier le produit", le champ Nom du produit affiche littéralement
"Sandale En Cuir  Rf 209 Noir" (DEUX espaces entre "Cuir" et "Rf" — visible
dans le screenshot, le curseur/sélection le confirme). Si `normStr()` ne
collapse pas TOUS les types d'espaces (espace insécable copié-collé depuis
Excel/Sheets, tabulation, espace double) vers un espace unique avant
comparaison, "Sandale En Cuir  Rf 209 Noir" (catalogue) et
"Sandale En Cuir Rf 209 Noir 42" (commande, espace simple) ne matcheront
JAMAIS, même avec la logique de variante correcte.

Dans server/services/variants.ts, retrouver `normStr` et s'assurer qu'elle
fait bien ceci (renforcer si besoin) :

```ts
export function normStr(s: string): string {
  return (s || '')
    .normalize('NFKD')                    // décompose les accents
    .replace(/[\u0300-\u036f]/g, '')      // retire les diacritiques
    .replace(/[\u00A0\u2000-\u200B\t\n\r]/g, ' ') // espaces insécables/unicode/tabs → espace normal
    .toLowerCase()
    .replace(/[^a-z0-9]+/g, ' ')          // toute ponctuation/espace multiple → un seul espace
    .trim()
    .replace(/\s+/g, ' ');
}
```

Le point clé si ce n'était pas déjà le cas : `\u00A0` (espace insécable) et
la plage `\u2000-\u200B` (espaces unicode divers) doivent être neutralisés
AVANT le `replace(/[^a-z0-9]+/g, ' ')`, sinon un espace "spécial" invisible
peut ne pas être traité comme un séparateur par certains moteurs regex.

ÉTAPE 2 — Endpoint de diagnostic temporaire (à garder ou supprimer après

```ts
app.get("/api/debug/resolve-product-name", requireAuth, requireAdmin, async (req: any, res: any) => {
  const storeId = req.user.storeId;
  const rawName = String(req.query.name || '');
  const storeProducts = await storage.getProductsByStore(storeId);
  const allVariants = await db.select().from(productVariants).where(eq(productVariants.storeId, storeId));
  const variantsByProduct = new Map<number, { name: string }[]>();
  for (const v of allVariants) {
    if (!variantsByProduct.has(v.productId)) variantsByProduct.set(v.productId, []);
    variantsByProduct.get(v.productId)!.push({ name: v.name });
  }
  const storeProductsWithVariants = storeProducts.map(p => ({ ...p, variants: variantsByProduct.get(p.id) || [] }));
  const result = resolveProductId(rawName, storeProductsWithVariants);
  res.json({
    rawName,
    normalizedRawName: normStr(rawName),
    result,
    candidateProductNames: storeProducts.map(p => ({ id: p.id, name: p.name, normalized: normStr(p.name) })),
  });
});
```

`normalizedRawName` correspond ou pas à un `normalized` de
`candidateProductNames`, ce qui confirmera (ou infirmera) le diagnostic de
