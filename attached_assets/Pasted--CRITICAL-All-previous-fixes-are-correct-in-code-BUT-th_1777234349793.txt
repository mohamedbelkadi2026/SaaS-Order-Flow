Step 7 — Add a "diagnostic" endpoint (one-time, useful for debugging)
Add this admin-only route in server/routes.ts:
tsapp.get("/api/_debug/distribution/:magasinId", requireAuth, async (req, res) => {
  const id = Number(req.params.magasinId);
  const [magasin] = await db.select().from(stores).where(eq(stores.id, id)).limit(1);
  const settings = await db.select().from(storeAgentSettings).where(eq(storeAgentSettings.storeId, magasin.ownerId ? /* parent */ id : id));
  const sinceEpoch = magasin?.distributionEpoch ?? new Date(0);
  const counts = await db.select({
    agentId: orders.assignedToId,
    count: sql<number>`count(*)`,
  }).from(orders)
    .where(and(eq(orders.magasinId, id), gte(orders.createdAt, sinceEpoch)))
    .groupBy(orders.assignedToId);

  res.json({
    magasin: { id: magasin?.id, name: magasin?.name, distributionMethod: magasin?.distributionMethod, distributionEpoch: magasin?.distributionEpoch, agentIds: magasin?.agentIds },
    perAgentSettings: settings.map(s => ({ agentId: s.agentId, leadPercentage: s.leadPercentage, roleInStore: s.roleInStore })),
    countsSinceEpoch: counts,
  });
});
Call it: GET /api/_debug/distribution/<magasinId>. The response tells you in one shot whether the magasin is configured for pourcentage, what each agent's target % is, and how many orders each has received since the epoch. If anything is off here, the UI/save layer is broken; if everything looks right but distribution still skews, the engine is broken.
