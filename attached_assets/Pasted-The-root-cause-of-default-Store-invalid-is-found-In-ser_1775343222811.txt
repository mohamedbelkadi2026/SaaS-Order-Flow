The root cause of "default Store invalid" is found. In `server/storage.ts`, the `pickFields` function inside `getAccountForShipping` uses `(a as any).carrierStoreName` which works at runtime — BUT the interface on line 75 declares the return type without `carrierStoreName`, so TypeScript may be stripping it in some build configurations.

Make these exact changes:

**Fix 1 — `server/storage.ts` line 75, update the interface:**
Find:
```typescript
getAccountForShipping(storeId: number, provider: string, city?: string): Promise<{ apiKey: string; apiSecret?: string; apiUrl?: string } | null>;
```
Replace with:
```typescript
getAccountForShipping(storeId: number, provider: string, city?: string): Promise<{ apiKey: string; apiSecret?: string; apiUrl?: string; carrierStoreName?: string; networkId?: number } | null>;
```

**Fix 2 — `server/storage.ts`, in `pickFields` inside `getAccountForShipping`, stop using `(a as any)`:**
Find:
```typescript
const pickFields = (a: typeof active[0]) => ({
  apiKey:           a.apiKey,
  apiSecret:        a.apiSecret       ?? undefined,
  apiUrl:           a.apiUrl          ?? undefined,
  carrierStoreName: (a as any).carrierStoreName ?? undefined,
});
```
Replace with:
```typescript
const pickFields = (a: typeof active[0]) => ({
  apiKey:           a.apiKey,
  apiSecret:        a.apiSecret        ?? undefined,
  apiUrl:           a.apiUrl           ?? undefined,
  carrierStoreName: a.carrierStoreName ?? undefined,
  networkId:        (a.settings as any)?.networkId ? Number((a.settings as any).networkId) : undefined,
});
```

const resolvedStore = storeName;
