FIX — Méthode de répartition: clarify Auto, allow flexible percentages, link agent to products, link agent to regions
What I want
1. Auto = pure round-robin. 1 commande / agent / round, regardless of how many agents (3, 5, 10…). Already works in code — just make UI explanation clear.
2. Pourcentage = flexible ratios that work however the user sets them. The UI currently scolds the user for not summing to 100% — that's wrong. The engine already normalizes. The right behavior: ratios are RELATIVE weights. 70/15/15 → A gets 70% of leads, B gets 15%, C gets 15%. 50/25/25 → 2/1/1. 5/2/2 → also 50/20/20 (engine normalizes). User shouldn't have to math their way to 100.
3. Produit = pick which products this agent handles. Orders containing those products go to this agent only.
4. Région = pick which regions this agent handles. Orders to those regions go to this agent only. Region → city mapping is already in server/storage.ts (REGION_CITY_MAP) — wire the UI to use it.
