CAUSE PROBABLE
La feature de tarifs par ville (carrier_city_pricing) ne s'applique qu'aux
NOUVELLES commandes (au ship-time / au webhook "delivered"). Toutes les
commandes Express Coursier déjà en base (livrées avant le déploiement, ou
mises à jour par un ancien webhook avant que la table de prix existe) ont
gardé shippingCost = 0 / null, donc la colonne "FRAIS DE LIVRAISON" du tableau
des commandes (celle qui affiche le petit logo transporteur + le montant)
n'a rien à afficher.

2) BACKFILL — remplir shippingCost pour TOUTES les commandes Express Coursier
existantes, quel que soit leur statut (pas seulement "delivered"/final).

Ajouter un endpoint one-shot dans server/routes.ts, près des autres routes
`/api/carriers/expresscoursier/...` :

```ts
app.post("/api/carriers/expresscoursier/backfill-shipping-cost", requireAuth, requireAdmin, async (req: any, res: any) => {
  const storeId = req.user.storeId;
  // Toutes les commandes de ce store expédiées via Express Coursier
  // (carrierName OU shippingProvider — les deux champs sont utilisés selon l'endroit
  // du code, voir orders.carrierName / orders.shippingProvider dans shared/schema.ts),
  // peu importe le statut, où shippingCost est 0/null.
  const orders = await storage.getOrdersByStoreAndCarrier(storeId, 'expresscoursier');
  // Si getOrdersByStoreAndCarrier n'existe pas encore, l'ajouter dans storage.ts :
  //   select * from orders where store_id = $1
  //   and (lower(carrier_name) in ('expresscoursier','express coursier','olivraison')
  //        or lower(shipping_provider) in ('expresscoursier','express coursier','olivraison'))

  let updated = 0, skippedNoCity = 0, usedDefault = 0;
  for (const order of orders) {
    if (order.shippingCost && order.shippingCost > 0) continue; // déjà rempli, ne pas écraser
    if (!order.customerCity) { skippedNoCity++; continue; }

    const cityFee = await storage.getCarrierCityPrice(storeId, 'expresscoursier', order.customerCity);
    const fee = cityFee ?? (EC_DEFAULT_CITY_PRICE_DH * 100); // fallback 35 DH si ville pas dans la table
    if (!cityFee) usedDefault++;

    await storage.updateOrder(order.id, { shippingCost: fee });
    updated++;
  }

  res.json({ message: `${updated} commandes mises à jour`, updated, usedDefault, skippedNoCity, total: orders.length });
});
```

Importer `EC_DEFAULT_CITY_PRICE_DH` depuis `./seed-data/ec-city-pricing` en
haut du fichier.
