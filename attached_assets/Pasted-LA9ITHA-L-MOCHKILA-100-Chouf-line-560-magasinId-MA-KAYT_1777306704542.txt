FIX — Dashboard "Tous les Magasins" filter is silently ignored by the API
Root cause (1 line in 1 file)
server/routes.ts line 560 — the destructure of req.query doesn't include magasinId:
tsconst { city, productId, source, dateFrom, dateTo, shippingProvider, utmSource, utmCampaign } = req.query as Record<string, string>;
//   ❌ no magasinId here
The client correctly sends ?magasinId=37 (line 103 of dashboard.tsx), but the server never reads it. So the orders array is never filtered by magasin and the stats stay identical regardless of which magasin you pick — exactly what's happening in my screenshot (12 commandes total whether I pick "Tous", "Anakio", or "Promomarkett").
Fix — server/routes.ts /api/stats/filtered endpoint
Replace line 560 with:
tsconst { city, productId, source, dateFrom, dateTo, shippingProvider, utmSource, utmCampaign, magasinId } = req.query as Record<string, string>;
Then add the filter clause among the others (immediately after the utmCampaign filter, around line 594):
tsif (magasinId && magasinId !== 'all') {
  const mid = Number(magasinId);
  allOrders = allOrders.filter(o => (o as any).magasinId === mid);
}
That's the entire fix for the dashboard. Two added lines.

For each one, add magasinId to the destructure AND add the matching .filter() clause. If they're already correct, leave them alone — but check explicitly, don't assume.
