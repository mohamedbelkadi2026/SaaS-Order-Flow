FIX 2 — Packaging cost is ignored in the global profit (Dashboard "Profit Net" AND
"Rentabilité Avancée" show Coût emballage = 0).

Root cause: profit is computed from the STORE-level store.packagingCost (= 0) instead of the
PER-PRODUCT settings.profitDefaults.coutEmballage. Note: the per-product profitability endpoint
(GET /api/products/profitability) already does it correctly using
item.productSettings.profitDefaults.coutEmballage — make the other two match it.

IMPORTANT UNIT RULE: in getAdminProfitSummary() and in the /api/stats handler, all money is in
CENTIMES (totalPrice, productCost, shippingCost are centimes; agent commission is rate*100).
But coutEmballage is stored in DH. So when adding packaging there you MUST multiply by 100
(DH → centimes). Packaging is per ORDER ("DH / commande"), so add it once per delivered order,
NOT multiplied by quantity.

2a) server/storage.ts → getAdminProfitSummary():
   - Stop using storePackaging (store[0].packagingCost) for packaging.
   - Build a map productId → coutEmballage(DH) from the store products' settings.profitDefaults.coutEmballage.
   - For each delivered order, compute its packaging = SUM over its order_items of
     (coutEmballage of that item's product) , in DH, then ×100 to centimes.
   - Replace `packagingCostTotal += Number(storePackaging)` with this per-order packaging (×100).
   - In the byBuyer loop, replace the `- Number(storePackaging)` term with the SAME per-order
     packaging value (×100) for that order.

2b) server/routes.ts → the GET /api/stats handler (the dashboard stats):
   - In the loop over delivered orders, replace `totalPackaging += storePackagingCost` with the
     per-order packaging computed from each order's items' product coutEmballage (DH) ×100.
   - Build the productId→coutEmballage map once before the loop (from storeProducts settings),
     and an orderId→items map, so you can sum per order.

on the Dashboard, in Rentabilité Avancée, and in the per-product Profitability table.
