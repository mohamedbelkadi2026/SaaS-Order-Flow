FEATURE — Tarifs de livraison automatiques par ville pour Express Coursier
(+ tout autre transporteur sans coût par ville via API)

CONTEXTE
Aujourd'hui, `carrierAccounts.deliveryFee` (shared/schema.ts) est un SEUL frais
fixe par compte transporteur, appliqué à TOUTES les villes (voir le fallback
"static fee" dans le webhook générique, server/routes.ts, bloc
"Auto-set shippingCost when order is delivered", ~ligne 4334). Pour Digylog on
récupère le vrai coût par ville via `getDigylogDeliveryCost()` (API), mais
Express Coursier n'a AUCUN endpoint qui retourne le prix de livraison — donc
toutes les commandes EC reçoivent le même `deliveryFee` statique, peu importe
la ville, ce qui fausse le calcul de profit.

Objectif : créer une table de tarifs par ville par transporteur, la pré-remplir
avec les vrais prix Express Coursier (calculés à partir de l'historique réel de
11 203 commandes), l'utiliser pour remplir `orders.shippingCost`
automatiquement (au ship + en fallback webhook + en backfill), et l'exposer
dans l'UI pour que je puisse la consulter/corriger.

────────────────────────────────────────────────────────────────────────────
ÉTAPE 1 — Schéma (shared/schema.ts)

Ajouter une nouvelle table, juste après `expressCoursierCities` :

```ts
// ─── Per-city delivery pricing (per carrier) ────────────────────────────────
// Fills orders.shippingCost automatically for carriers that don't return a
// real per-city cost via API (Express Coursier has no such endpoint — unlike
// Digylog's getDigylogDeliveryCost). One row per store+carrier+city.
export const carrierCityPricing = pgTable("carrier_city_pricing", {
  id:          serial("id").primaryKey(),
  storeId:     integer("store_id").notNull(),
  carrierName: text("carrier_name").notNull(),   // e.g. "expresscoursier"
  cityName:    text("city_name").notNull(),       // display name, as typed by the user
  cityNorm:    text("city_norm").notNull(),        // normalizeCityKey(cityName) — used for lookup
  priceDh:     integer("price_dh").notNull(),      // price in CENTIMES (same convention as shippingCost/deliveryFee — e.g. 3500 = 35.00 DH)
  source:      text("source").default("manual"),   // "manual" | "import_historique"
  createdAt:   timestamp("created_at").defaultNow(),
  updatedAt:   timestamp("updated_at").defaultNow(),
}, (table) => ({
  uniqueCityPerCarrier: uniqueIndex("carrier_city_pricing_unique")
    .on(table.storeId, table.carrierName, table.cityNorm),
}));
export type CarrierCityPricing = typeof carrierCityPricing.$inferSelect;
```

Run the migration (`npm run db:push` or the project's existing drizzle push
script — check package.json).

ÉTAPE 2 — Storage layer (server/storage.ts)

Add next to the existing `expressCoursierCities` helpers:

```ts
async getCarrierCityPricing(storeId: number, carrierName: string) {
  return db.select().from(carrierCityPricing)
    .where(and(eq(carrierCityPricing.storeId, storeId), eq(carrierCityPricing.carrierName, carrierName)));
}

// city price lookup used at ship-time / webhook fallback — returns centimes or null
async getCarrierCityPrice(storeId: number, carrierName: string, cityRaw: string): Promise<number | null> {
  const norm = normalizeCityKey(cityRaw || "");
  if (!norm) return null;
  const [row] = await db.select().from(carrierCityPricing)
    .where(and(
      eq(carrierCityPricing.storeId, storeId),
      eq(carrierCityPricing.carrierName, carrierName),
      eq(carrierCityPricing.cityNorm, norm),
    ));
  return row ? row.priceDh : null;
}

async upsertCarrierCityPrice(storeId: number, carrierName: string, cityName: string, priceDh: number, source = "manual") {
  const norm = normalizeCityKey(cityName);
  const existing = await db.select().from(carrierCityPricing)
    .where(and(eq(carrierCityPricing.storeId, storeId), eq(carrierCityPricing.carrierName, carrierName), eq(carrierCityPricing.cityNorm, norm)));
  if (existing.length) {
    await db.update(carrierCityPricing).set({ priceDh, cityName, source, updatedAt: new Date() }).where(eq(carrierCityPricing.id, existing[0].id));
  } else {
    await db.insert(carrierCityPricing).values({ storeId, carrierName, cityName, cityNorm: norm, priceDh, source });
  }
}
```

Import `normalizeCityKey` from `./services/city-aliases` at the top of
storage.ts if not already imported.

ÉTAPE 4 — Endpoint d'import (à appeler UNE SEULE FOIS après déploiement)

In server/routes.ts, add near the other `/api/carriers/expresscoursier/...`
routes:

```ts
app.post("/api/carriers/expresscoursier/import-city-pricing", requireAuth, requireAdmin, async (req: any, res: any) => {
  const storeId = req.user.storeId;
  const { EC_CITY_PRICING_SEED } = await import("./seed-data/ec-city-pricing");
  let count = 0;
  for (const [city, priceDh] of EC_CITY_PRICING_SEED) {
    await storage.upsertCarrierCityPrice(storeId, "expresscoursier", city, priceDh * 100, "import_historique");
    count++;
  }
  res.json({ message: `${count} tarifs de ville importés pour Express Coursier`, count });
});
```

ÉTAPE 5 — Utiliser le tarif automatiquement (3 endroits)

5a) AU MOMENT DE L'EXPÉDITION — server/routes.ts, dans le bloc d'expédition
Express Coursier (~ligne 1723, là où le `city id` est résolu avant l'appel API
EC). Juste après un envoi réussi et avant/juste après avoir set
`shippingProvider`/`carrierName`, si `order.shippingCost` est 0, faire :

```ts
if (!order.shippingCost || order.shippingCost === 0) {
  const cityPrice = await storage.getCarrierCityPrice(storeId, 'expresscoursier', order.customerCity || '');
  const fee = cityPrice ?? (EC_DEFAULT_CITY_PRICE_DH * 100); // fallback 35 DH si ville absente de la table
  await storage.updateOrder(order.id, { shippingCost: fee });
}
```

5b) FALLBACK WEBHOOK — server/routes.ts, bloc "Auto-set shippingCost when
order is delivered" (~ligne 4334). Actuellement il essaie
`getDigylogDeliveryCost` puis retombe sur `acct.deliveryFee` (fixe). AJOUTER
un palier intermédiaire "prix par ville" AVANT le fallback statique :

```ts
if (cost && cost > 0) {
  await storage.updateOrder(order.id, { shippingCost: cost });
  console.log(`[DeliveryFee] Order #${(order as any).orderNumber} delivered — shippingCost=${cost} (per-city API)`);
} else {
  // NOUVEAU : tarif par ville stocké (Express Coursier & tout transporteur sans API de coût)
  const cityFee = await storage.getCarrierCityPrice((order as any).storeId, carrierName, (order as any).customerCity || '');
  if (cityFee && cityFee > 0) {
    await storage.updateOrder(order.id, { shippingCost: cityFee });
    console.log(`[DeliveryFee] Order #${(order as any).orderNumber} delivered — shippingCost=${cityFee} (per-city table)`);
  } else {
    const fee = (acct as any)?.deliveryFee || 0;
    if (fee > 0) {
      await storage.updateOrder(order.id, { shippingCost: fee });
      console.log(`[DeliveryFee] Order #${(order as any).orderNumber} delivered — shippingCost=${fee} (static fee)`);
    }
  }
}
```

5c) BACKFILL — server/routes.ts, le job "FIX-COST" (~ligne 8903, orders avec
`shippingCost === 0` en statut final). Même logique : essayer
`storage.getCarrierCityPrice(storeId, carrierName, order.customerCity)` avant
tout fallback statique, pour corriger aussi les vieilles commandes EC déjà en
base.

VÉRIFICATION
- Après l'import, une commande Express Coursier vers Casablanca doit recevoir
  shippingCost = 2000 (20 DH), vers une grande ville (Marrakech, Tanger, Rabat,
  Fes, Agadir, Meknes, Kenitra, Sale, Oujda, El jadida, Mohammedia, Tetouan,
  Safi, Beni mellal, Temara) shippingCost = 3000 (30 DH), et vers une ville
  secondaire/rurale shippingCost = 3500 (35 DH, tarif par défaut) — PAS le
  même deliveryFee fixe qu'avant pour toutes les villes.
- Une ville absente de la table (nouvelle ville jamais vue) reçoit 3500
  (fallback 35 DH), pas 0.
- Le Profit Analyzer (server/services/profit.ts, qui lit déjà
  `order.shippingCost` directement — AUCUNE modification nécessaire là-bas)
  reflète immédiatement le vrai coût par ville dans le netProfit.
- Lancer le job de backfill sur les commandes EC déjà livrées avec
