FIX — When a carrier webhook arrives for a tracking number the platform doesn't recognize, AUTO-CREATE the order from the carrier's data
My situation
I have ~100 orders that I shipped to Digylog/Maystro/Ameex 3 days ago, BEFORE the platform's webhook integration was set up. Now the carrier API has all the data (customer name, phone, address, city, price, current status) but the platform's database doesn't even know these orders exist. So:

Webhook arrives with tracking=SB9A3675N, phone=0707835334, status=En cours de réception au hub de livraison
Platform tries to match by tracking → not found
Platform tries by ref → not found
Webhook gets logged as "No Match" and abandoned
The order never appears in the platform, even though the carrier knows everything

What I want: the webhook itself should create the order in the platform, pulling missing fields (name, address, city, price) directly from the carrier's API. From that point forward, every status update flows through normally.

Files to change

server/routes.ts — extend processCarrierWebhook with auto-create fallback
server/services/carrier-service.ts — add a fetchOrderDetails(provider, trackingNumber, account) helper that returns full customer data, not just status
server/storage.ts — add createOrderFromCarrier(...) helper

Step 2 — createOrderFromCarrier in storage
In server/storage.ts, add a helper that creates a minimal order from carrier-provided data. Default agent assignment goes through normal getNextAgent so multi-magasin distribution still works.
tsasync createOrderFromCarrier(params: {
  storeId: number;
  magasinId?: number | null;
  provider: string;
  trackingNumber: string;
  customerName: string;
  customerPhone: string;
  customerAddress?: string;
  customerCity?: string;
  totalPrice?: number;        // centimes
  shippingCost?: number;      // centimes
  status?: string;            // already mapped to internal status
  rawStatus?: string;
}): Promise<Order> {
  // Use the tracking number as the order_number to keep it unique and obvious in the UI
  const orderNumber = `EXT-${params.trackingNumber}`;
  const assignedToId = await this.getNextAgent(
    params.storeId, params.magasinId ?? null, undefined, params.customerCity
  );

  const [created] = await db.insert(orders).values({
    storeId: params.storeId,
    magasinId: params.magasinId ?? null,
    orderNumber,
    customerName: params.customerName || 'Client (importé)',
    customerPhone: params.customerPhone || '',
    customerAddress: params.customerAddress || '',
    customerCity: params.customerCity || '',
    totalPrice: params.totalPrice ?? 0,
    shippingCost: params.shippingCost ?? 0,
    trackNumber: params.trackingNumber,
    shippingProvider: params.provider,
    status: params.status || 'Attente De Ramassage',
    commentStatus: params.rawStatus || '',
    source: `${params.provider}_webhook`,
    assignedToId,
    createdAt: new Date(),
    updatedAt: new Date(),
  } as any).returning();
  return created;
}

Step 4 — Backfill button: "Importer toutes les commandes Digylog"
For my 100 historical orders that arrived BEFORE I set up the webhook, add a manual one-shot endpoint that pulls the full list from the carrier and creates them all. This is different from "sync statuses for existing orders" — it's "import orders that don't exist yet".
tsapp.post("/api/shipping/:provider/import-historical", requireAuth, requireActiveSubscription, async (req, res) => {
  try {
    const provider = req.params.provider.toLowerCase();
    const storeId = req.user!.storeId!;
    const { since } = req.body || {};   // optional ISO date — only import orders created after this date

    const accounts = await storage.getCarrierAccounts(storeId, provider);
    const account = accounts[0];
    if (!account) return res.status(400).json({ message: `Aucun compte ${provider} configuré.` });

    // Carrier-specific list endpoint. For Digylog: GET /api/v2/seller/orders?from=YYYY-MM-DD
    const { listOrdersFromCarrier } = await import("./services/carrier-service");
    const carrierOrders = await listOrdersFromCarrier(provider, account, { since });
    if (!carrierOrders || carrierOrders.length === 0) {
      return res.json({ created: 0, skipped: 0, message: "Aucune commande à importer." });
    }

    const magasins = await storage.getStoresByOwner(req.user!.id);
    const magasinId = magasins.length === 1 ? magasins[0].id : null;

    let created = 0, skipped = 0;
    for (const co of carrierOrders) {
      // Skip if already in platform (matched by tracking)
      const existing = await storage.getOrderByTrackingNumber(storeId, co.trackingNumber);
      if (existing) { skipped++; continue; }

      await storage.createOrderFromCarrier({
        storeId,
        magasinId,
        provider,
        trackingNumber: co.trackingNumber,
        customerName:    co.customerName,
        customerPhone:   co.customerPhone,
        customerAddress: co.customerAddress,
        customerCity:    co.customerCity,
        totalPrice:      co.totalPrice,
        shippingCost:    co.shippingCost,
        status:          co.status,
        rawStatus:       co.rawStatus,
      });
      created++;
    }
    res.json({ created, skipped, total: carrierOrders.length });
  } catch (err: any) {
    res.status(500).json({ message: err.message });
  }
});
In carrier-service.ts, add listOrdersFromCarrier:
tsexport async function listOrdersFromCarrier(
  provider: string,
  account: any,
  options?: { since?: string }
): Promise<Array<any>> {
  if (provider.toLowerCase() === 'digylog') {
    const apiKey = (account as any).apiKey;
    const base = ((account as any).apiUrl || 'https://api.digylog.com/api/v2/seller').replace(/\/+$/, '');
    const params: any = {};
    if (options?.since) params.from = options.since;

    const resp = await axios.get(`${base}/orders`, {
      headers: { 'Authorization': `Bearer ${apiKey}`, 'Accept': 'application/json' },
      params,
      timeout: 30000,
      httpsAgent: SSL_AGENT,
    });

    const list = resp.data?.data || resp.data?.orders || resp.data || [];
    return list.map((o: any) => ({
      trackingNumber: o.traking || o.tracking || o.code,
      customerName:   o.name      || '',
      customerPhone:  o.phone     || '',
      customerAddress: o.address  || '',
      customerCity:   o.city      || '',
      totalPrice:     o.price        ? Math.round(parseFloat(o.price) * 100)        : 0,
      shippingCost:   o.deliveryCost ? Math.round(parseFloat(o.deliveryCost) * 100) : 0,
      rawStatus:      o.status || '',
      // Map raw status to internal status using the same logic as trackDigylogShipment
      status: mapDigylogStatus(o.status || ''),
    }));
  }
  return [];
}

function mapDigylogStatus(rawText: string): string {
  const rawLow = rawText.toLowerCase();
  if (
    rawLow === 'livré' || rawLow === 'livre' || rawLow === 'livrée' ||
    rawLow === 'remis au client' || rawLow.includes('distribu')
  ) return 'delivered';
  if (rawLow.includes('retour') && !rawLow.includes('en cours')) return 'retourné';
  if (rawLow.includes('refus') || rawLow.includes('annul')) return 'refused';
  if (rawLow.includes('ramass') || rawLow.includes('attente')) return 'Attente De Ramassage';
  return 'in_progress';
}
