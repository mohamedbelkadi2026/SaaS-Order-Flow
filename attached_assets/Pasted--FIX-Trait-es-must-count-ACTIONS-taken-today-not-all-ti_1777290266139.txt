🎯 FIX — "Traitées" must count ACTIONS taken today (not all-time orders) + show per-magasin
My understanding of how the platform should behave
There are TWO separate concepts that are currently conflated:
1. Distribution math (getNextAgent): When I change percentages, the engine should rebalance from the moment of the change forward. Old orders should NOT influence the new distribution. This already works correctly via distributionEpoch — the engine reads windowStart = magasin.distributionEpoch and counts only orders created after it. Good.
2. UI "Performance" column on the Team page: The "Traitées" number must show how many orders the agent took action on today (changed status, confirmed, etc.) — not the all-time count of orders ever assigned to them. The current code shows count(*) of all assigned orders ever, which is misleading and inflates as the platform ages.
These two systems must NOT share the same counter — that was the source of all the confusion. Distribution uses its own epoch-windowed count internally; the UI display uses an action-based daily count.

Step 3 — Rewrite getAgentPerformance to count today's actions, scoped per magasin
In server/storage.ts line 1610, replace the function:
tsasync getAgentPerformance(
  storeId: number,
  options?: { magasinId?: number | null; date?: string }   // date in YYYY-MM-DD
): Promise<{ agentId: number; total: number; confirmed: number; delivered: number; cancelled: number }[]> {
  // Default window = today (00:00 → 23:59:59 in server time)
  const dayStart = new Date();
  if (options?.date) {
    const [y, m, d] = options.date.split('-').map(Number);
    dayStart.setFullYear(y, m - 1, d);
  }
  dayStart.setHours(0, 0, 0, 0);
  const dayEnd = new Date(dayStart);
  dayEnd.setDate(dayEnd.getDate() + 1);

  const conds: any[] = [
    eq(orders.storeId, storeId),
    sql`${orders.lastActionBy} IS NOT NULL`,
    gte(orders.lastActionAt, dayStart),
    lt(orders.lastActionAt, dayEnd),
  ];
  if (options?.magasinId) conds.push(eq(orders.magasinId, options.magasinId));

  const result = await db.select({
    agentId: orders.lastActionBy,                      // ← group by who ACTED, not who's assigned
    total: count(),
    confirmed: sql<number>`count(*) filter (where ${orders.status} in ('confirme', 'expédié', 'delivered', 'refused', 'Attente De Ramassage', 'in_progress', 'retourné'))`,
    delivered: sql<number>`count(*) filter (where ${orders.status} = 'delivered')`,
    cancelled: sql<number>`count(*) filter (where ${orders.status} in ('Annulé (fake)', 'Annulé (faux numéro)', 'Annulé (double)'))`,
  }).from(orders)
    .where(and(...conds))
    .groupBy(orders.lastActionBy);

  return result.map(r => ({
    agentId: r.agentId!,
    total: Number(r.total),
    confirmed: Number(r.confirmed),
    delivered: Number(r.delivered),
    cancelled: Number(r.cancelled),
  }));
}
Update the interface signature on line 127 to match the new options arg.

    },
  });
}
