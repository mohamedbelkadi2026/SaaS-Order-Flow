FEATURE — "Confirmé Reporté" badge shows count of orders scheduled for today/tomorrow
My problem
After adding the "Confirmé Reporté" status, agents have no visual cue when an upcoming scheduled date is approaching. If a customer asked to be re-confirmed on May 5 and an agent forgets, the order sits silently until the cron auto-promotes it on the morning of the 5th — too late to call the client and verify.
I want a red badge on the "Confirmé Reporté" sidebar item that shows how many orders are due for re-confirmation by tomorrow. So on May 4 evening / May 5 morning, the agent sees "Confirmé Reporté [3]" and knows there are 3 customers to call before shipping the next day.
Logic

Today = current Casablanca date (UTC+1)
"Due soon" = orders with status='confirme_reporte' AND scheduled_for <= today+1 (i.e. today or tomorrow)
Show badge only when count > 0

This means the badge appears the day before the scheduled delivery date, giving the agent 24 hours to call and re-confirm with the client.

Step 5 — Sort the tab by scheduledFor ASC so soonest-due comes first
In the page's data fetching, when urlStatus === 'confirme_reporte', add a sort:
tsxconst orders = useMemo(() => {
  let list = (data?.orders || []) as any[];

  if (urlStatus === 'confirme_reporte') {
    list = [...list].sort((a, b) => {
      const da = a.scheduledFor ? new Date(a.scheduledFor).getTime() : Infinity;
      const db = b.scheduledFor ? new Date(b.scheduledFor).getTime() : Infinity;
      return da - db;  // soonest first
    });
  }

  return list;
}, [data, urlStatus]);
The agent opens the tab and sees the most-urgent reconfirmations at the top.
Step 6 — Color-code rows by urgency
In the row render, when on the confirme_reporte tab, add a left border to indicate urgency:
tsxconst getUrgencyClass = (order: any): string => {
  if (urlStatus !== 'confirme_reporte' || !order.scheduledFor) return '';
  const today = new Date();
  today.setHours(0, 0, 0, 0);
  const target = new Date(order.scheduledFor);
  target.setHours(0, 0, 0, 0);
  const daysUntil = Math.round((target.getTime() - today.getTime()) / 86400000);
  if (daysUntil <= 0) return 'border-l-4 border-red-500';      // due today or overdue (about to be auto-promoted)
  if (daysUntil <= 1) return 'border-l-4 border-amber-500';    // due tomorrow
  if (daysUntil <= 3) return 'border-l-4 border-yellow-300';   // due this week
  return 'border-l-4 border-transparent';                       // farther than 3 days
};

<TableRow className={getUrgencyClass(order)}>
  ...
</TableRow>
Combined with the sort, the agent sees a clear visual gradient: red urgent → amber soon → yellow this week → no border later.
