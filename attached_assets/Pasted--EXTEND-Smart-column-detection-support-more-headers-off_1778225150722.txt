🚀 EXTEND — Smart column detection: support more headers (offer, fullname, note, UTM, product ID)
My problem
The smart-detection script I just deployed handles Nom, Téléphone, Adresse, Ville, Produit, Prix, Quantité. But many of my real sheets have additional columns I want imported:
| Product | No Label Offre | No Label fullname | No Label phone | No Label address | City | Price | Note | orders | utm_campaign | utm_source | Product ID |
The "No Label" prefix is a quirk from a tool that exported the sheet (Tally? Google Forms?). My existing script ignores those columns entirely. I want:

fullname recognized as the customer's name (currently only "nom" / "name" work)
No Label X prefix tolerated everywhere
Note column → fills order.comment
utm_source + utm_campaign → fills order.utmSource and order.utmCampaign
Product ID → fills a new order.ameexProductId so Ameex shipping can use it
Offer / Offre → fills a new order.offerName (just metadata, shown in admin)
Status / orders column → already detected as the status column (don't try to use its value as an order status, just write SENT/EN ATTENTE/... to it)

The user shouldn't have to change any of their headers.

The trick that handles "No Label X" prefixes is in normalizeHeader — it strips non-alphanumeric junk. But to make it robust against the no label prefix specifically, update normalizeHeader:
jsfunction normalizeHeader(s) {
  return String(s || '')
    .toLowerCase()
    .normalize('NFD').replace(/[\u0300-\u036f]/g, '')   // strip accents
    .replace(/^no\s*label\s*/i, '')                      // strip 'No Label' prefix
    .replace(/^col(onne|umn)?\s*/i, '')                  // strip 'Col' / 'Colonne' / 'Column' prefix
    .replace(/[^a-z0-9\u0600-\u06ff]+/g, ' ')
    .trim();
}
So "No Label fullname" becomes "fullname" → matches name alias → column detected.
