The Digylog API requires additional fields in the order payload that we are not sending.

Looking at the Digylog seller dashboard "Preferences Configuration", orders need:
- Store (already handled via carrierStoreName)
- Pickup Point (agence/hub where Digylog picks up parcels)
- Storage Point / Fulfilment Center (FC id)

These correspond to Digylog API fields:
- `network` = the network/hub id (Pickup Point → GET /networks)
- `fc` = fulfilment center id (Storage Point → GET /fc) — only needed if mode=2

The real issue: our payload sends `network: 1` hardcoded, but the user's actual network ID from their Digylog account may be different.

**Fix 2 — Add server endpoint to fetch Digylog networks**

In `server/routes.ts`, add:
```typescript
app.get("/api/carrier-accounts/digylog/networks", requireAuth, async (req, res) => {
  const { token, apiUrl } = req.query as { token?: string; apiUrl?: string };
  if (!token) return res.status(400).json({ message: "token requis" });
  
  const cleanToken = token.replace(/[\r\n\t\x00-\x1F\x7F]/g, "").trim();
  const baseUrl = (apiUrl || "https://api.digylog.com/api/v2/seller").replace(/\/+$/, "");
  
  try {
    const axiosLib = (await import("axios")).default;
    const resp = await axiosLib.get(`${baseUrl}/networks`, {
      headers: {
        Authorization: `Bearer ${cleanToken}`,
        Accept: "application/json",
        Referer: "https://apiseller.digylog.com",
        Origin: "https://apiseller.digylog.com",
      },
      timeout: 15_000,
      validateStatus: () => true,
    });
    if (resp.status !== 200) return res.status(resp.status).json({ message: `Digylog HTTP ${resp.status}`, raw: resp.data });
    const raw = resp.data;
    const list = Array.isArray(raw) ? raw : (raw?.data || []);
    res.json({ networks: list.map((n: any) => ({ id: n.id, name: n.name })) });
  } catch (err: any) {
    res.status(502).json({ message: "Impossible de contacter Digylog." });
  }
});
```
