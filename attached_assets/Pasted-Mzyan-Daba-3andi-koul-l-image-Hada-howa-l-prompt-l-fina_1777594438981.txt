FEATURE — Add "Confirmé Reporté" status with scheduled auto-promotion to "Confirmé"
My problem
Some customers say "yes, I want it, but ship it later — for example on May 6". Today I have to either:

Mark them "Confirmé" now → they ship today (wrong, we'll have stock issues)
Mark them "Pas de réponse" → wrong status, they did respond and they did agree

I want a new status "Confirmé Reporté" (Confirmed Postponed) where I save the agreed-upon delivery date, and the system automatically moves the order to "Confirmé" when that date arrives, ready for shipping.
What to build

New status string confirme_reporte in the database
New column scheduled_for (date) on orders to store when the order should auto-promote
New sidebar item "Confirmé Reporté" in Mes Commandes (between "Confirmés" and "Injoignables")

Confirmation modal where the agent picks a date when they hit "Confirmé Reporté"

Bulk-ship endpoint must still EXCLUDE confirme_reporte (already does, since it only allows confirme)
Stats endpoints (dashboard counts, agent performance) treat confirme_reporte as a confirmation for accounting purposes, but the order stays out of shipping queue until promoted.

Step 8 — Daily cron: promote scheduled orders to confirme when their date arrives
Create server/cron/promote-scheduled.ts:
tsimport { db } from '../db';
import { orders } from '@shared/schema';
import { and, eq, lte, sql } from 'drizzle-orm';

/**
 * Runs daily at 06:00 Casablanca time.
 * Finds every order with status='confirme_reporte' AND scheduled_for <= today
 * and promotes it to status='confirme', clearing scheduled_for.
 *
 * Idempotent: re-running on the same day produces zero new updates after
 * the first promotion.
 */
export async function promoteScheduledOrders(): Promise<{ promoted: number }> {
  const result = await db.update(orders)
    .set({
      status:       'confirme',
      scheduledFor: null,
      lastActionAt: new Date(),
    })
    .where(and(
      eq(orders.status, 'confirme_reporte'),
      lte(orders.scheduledFor, sql`CURRENT_DATE`),
    ))
    .returning({ id: orders.id, orderNumber: orders.orderNumber });

  if (result.length > 0) {
    console.log(`[CRON-PROMOTE] Promoted ${result.length} order(s) from confirme_reporte → confirme:`);
    result.forEach(o => console.log(`[CRON-PROMOTE]   - #${o.orderNumber} (id=${o.id})`));
  }

  return { promoted: result.length };
}
In server/index.ts, schedule it:
tsimport { promoteScheduledOrders } from './cron/promote-scheduled';

// ── Daily promotion of scheduled orders ───────────────────────────────────
// Runs every hour and only does work if it's currently 06:00 Casablanca local time.
// This catches the right moment regardless of when the server boots.
setInterval(async () => {
  const now = new Date();
  // Casablanca is UTC+1 year-round (no DST). Compute Casablanca hour from UTC.
  const casablancaHour = (now.getUTCHours() + 1) % 24;

  if (casablancaHour === 6 && now.getUTCMinutes() < 60) {
    // Mark we ran today using a module-level flag, to avoid double-running
    // if the interval fires twice in the 06:00-06:59 window.
    if ((global as any).__lastPromoteRunDay !== now.toISOString().slice(0, 10)) {
      (global as any).__lastPromoteRunDay = now.toISOString().slice(0, 10);
      try {
        await promoteScheduledOrders();
      } catch (err) {
        console.error('[CRON-PROMOTE] Failed:', err);
      }
    }
  }
}, 30 * 60 * 1000);  // Check every 30 minutes

// Also run once on boot, in case the server was down at 06:00.
// This catches any orders whose scheduled_for has already passed.
promoteScheduledOrders().catch(err => console.error('[CRON-PROMOTE] Boot run failed:', err));
