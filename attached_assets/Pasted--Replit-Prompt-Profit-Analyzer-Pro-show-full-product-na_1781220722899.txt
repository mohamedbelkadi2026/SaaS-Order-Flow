BUG 1 — Product names are cut off with "…" (ellipsis)
═══════════════════════════════════════════════════════════════════════
In the **Par Produit** results table the PRODUIT column truncates long names, e.g.
"CONFORT TOP BOSS EN CUIR B…", "Sandale En Cuir Evo Rf 0011 noir…". The user needs the
FULL name visible.

Do this:
1. Find the product-name cell(s). Look for the column header "PRODUIT" / "Produit" and the
   element that renders the product name string.
2. On that cell, REMOVE any of these classes if present: `truncate`, `text-ellipsis`,
   `overflow-hidden`, `whitespace-nowrap`, and any fixed/`max-w-[...]` width clamp.
3. REPLACE with: `whitespace-normal break-words` so the name wraps onto multiple lines and
   stays fully readable. Let the row height auto-grow.
4. Give the PRODUIT column more room than the others (make it the widest column — e.g. the
   largest flex value or `w-1/2`), since "CMD / UNITÉS" and "CA BRUT" are short.
5. Add `title={productName}` on the cell as a hover-tooltip fallback.
6. IMPORTANT: apply the SAME fix in EVERY place the product name is shown on this page —
   not only the "Par Produit" table, but also the Step 2 "Saisie des coûts" table and the
   saved-reports preview/list. Search the file for all product-name renders and de-truncate
   each one.
