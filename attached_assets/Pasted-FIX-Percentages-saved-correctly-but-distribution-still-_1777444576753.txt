FIX — Percentages saved correctly but distribution still round-robin → magasin's distribution_method is still 'auto'
Diagnosis
From your logs:
[{"magasinId":37,"leadPercentage":50},{"magasinId":42,"leadPercentage":100}]
Per-magasin percentages ARE saved correctly per agent.
From your screenshot:

Distribution alternates khawla / naima / fatima evenly = round-robin
50/25/25 ratio NOT respected

Root cause
The engine in server/storage.ts line 2004 reads distribution_method from the magasin row (Promomarkett, Anakio). It's still set to 'auto'. Until you flip it to 'pourcentage' per magasin, the engine never enters the percentage branch — it falls through to round-robin on line 2028.
Fix in 3 steps
Step 1 — Quick SQL one-shot (run NOW in Replit Database tab)
sql-- Set distribution method to 'pourcentage' for every magasin in your account
UPDATE public.stores
SET distribution_method = 'pourcentage'
WHERE owner_id IS NULL OR owner_id IN (SELECT id FROM public.users WHERE role IN ('owner', 'admin'));

-- Or target specific magasins if you only want some on percentage:
-- UPDATE public.stores SET distribution_method = 'pourcentage' WHERE id IN (37, 42);

-- Reset epoch + last assigned so the new distribution starts fresh
UPDATE public.stores
SET distribution_epoch = now(),
    last_assigned_agent_id = NULL
WHERE distribution_method = 'pourcentage';

-- Verify
SELECT id, name, distribution_method, distribution_epoch FROM public.stores;
After this, every new order on Promomarkett/Anakio will hit the percentage engine.

Step 3 — Add a quick toggle next to each magasin in the Team page
So admins can flip the method without leaving Team. Add a small action column:
tsx// In the magasinsOnAuto banner, add a "Activer maintenant" button per magasin:
{magasinsOnAuto.map((m: any) => (
  <div key={m.id} className="flex items-center justify-between text-xs gap-2 py-1">
    <span><span className="font-medium">{m.name}</span> — méthode : <code>auto</code></span>
    <Button
      size="sm"
      variant="default"
      onClick={async () => {

    </Button>
  </div>
))}
The PUT /api/magasins/:id endpoint already accepts distributionMethod (line 5094 in routes.ts). On success, the magasin's distribution_method changes to 'pourcentage', the epoch is bumped (via the existing route handler logic), and the next order goes through the % engine.
