Dashboard statistics are logically impossible: the "CONFIRMÉES" KPI (13) is LOWER than 
"EXPÉDIÉS" (36) for the same period, even though an order must be confirmed before it 
can be shipped. Also "EN COURS" shows 0 while 26 orders are in transit.

ROOT CAUSE (already diagnosed — do NOT re-investigate):
The stats code counts an order as "confirmed" only if its CURRENT status is in a 
hard-coded list of 8 statuses:
  ['confirme','confirme_reporte','expédié','delivered','refused','Attente De Ramassage','in_progress','retourné']
But once an order ships, its status becomes a carrier/transit status that is NOT in 
that list — e.g. 'En transit','En cours de livraison','Ramassé','Sorti pour livraison',
'En cours de réception au network','Arrivé au hub','En cours de distribution','Reçu',
'Programmé','En stock','Pris en charge','Collecté','Chargé','Confirmé par livreur',
'Tentative échouée','Retour Recu', and French delivered variants 'Livré'/'Livrée'. 
So confirmed shipped orders drop out of the confirmed count and land in "Autres" (49%), 
making CONFIRMÉES < EXPÉDIÉS. This same 8-status list is DUPLICATED in many places.

1) Create ONE shared helper (e.g. in shared/schema.ts or a new shared/order-status.ts)

     // A lead is "confirmed (cumulative)" once an agent confirms it; it STAYS 
     // confirmed through every downstream carrier/transit/delivered/returned status.
     export const NOT_CONFIRMED_STATUSES = new Set([
       'nouveau','rappel','Injoignable','boite vocale',
       'Annulé (fake)','Annulé (faux numéro)','Annulé (double)'
     ]);
     export function isConfirmedCumulative(status: string, trackNumber?: string | null): boolean {
       if (!status) return false;
       if (status.startsWith('Pas de réponse')) return false;
       if (NOT_CONFIRMED_STATUSES.has(status)) return false;
       return true; // everything else (confirme, confirme_reporte, all carrier/
                    // transit/delivered/returned statuses) counts as confirmed
     }
   Rationale: carrier statuses are numerous and keep growing; the pre-confirmation/
   cancelled lead statuses are few and stable. This guarantees confirmed >= shipped.
